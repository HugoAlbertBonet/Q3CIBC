# Batch pushingA — IBC-paper-faithful trials on Simulated Pushing (single target).
#
# The hyperparameters below mirror IBC's `pushing_states/mlp_ebm_langevin.gin`
# (Florence et al. 2021) wherever a direct mapping exists between IBC and
# Q3CIBC. See the fidelity report this batch was authored alongside for
# remaining structural gaps (action normalization, obs standardization).
#
# IBC-paper-matched knobs (held constant across all A* trials):
#   batch_size                   = 512        (IBC default)
#   training_steps               = 100_000    (IBC train_eval.num_iterations)
#   learning_rate                = 1e-3       (IBC train_eval.learning_rate)
#   goal_dist_tolerance          = 0.02       (env config; IBC goal_tolerance)
#   max_episode_steps            = 100        (env config; IBC BlockPush-v0)
#   frame_stack                  = 2          (IBC sequence_length=2)
#   q_network_kind               = resnet     (IBC MLPEBM.layers='ResNetPreActivation')
#   q_width                      = 128        (IBC MLPEBM.width=128)
#   q_depth                      = 8          ── 8 residual BLOCKS == 16 dense
#                                                layers, which is IBC's
#                                                MLPEBM.depth=16 (depth in the
#                                                paper counts Linear layers,
#                                                ours counts blocks; each block
#                                                contains 2 Linears).
#   top_k_control_points         = 8          (IBC ImplicitBCAgent.num_counter_examples=8)
#   gradient_penalty_weight      = 1.0        (IBC ImplicitBCAgent.add_grad_penalty=True)
#   gradient_penalty_margin      = 1.0        (IBC grad_penalty.grad_margin=1.0)
#   inference_langevin_iterations= 100        (IBC langevin_actions_given_obs.num_iterations=100)
#   langevin_lr_init             = 0.1        (Gap A/B closed: actions now
#                                                live in [-1, 1], so we use
#                                                IBC-paper scale lr/clip)
#   langevin_delta_clip          = 0.1
#   action space (in model + Q)  = [-1, 1]    Gap B closed: PushingDataset
#                                                normalizes actions, training
#                                                runs in [-1, 1], PushingSim
#                                                denormalizes to env scale.
#   obs normalization            = standardize Gap A closed: per-dim mean/std
#                                                computed from dataset (matches
#                                                IBC get_normalizers.py +
#                                                stats.compute_dataset_statistics).
#
# CP-generator side (Q3CIBC-specific — no IBC analog):
#   control_points               = 20         (CP cloud size; Q3CIBC concept)
#   cp_network_kind              = mlp        (IBC has no separate CP gen; keep small)
#   cp_width / cp_depth          = 256 / 2
#
# Wiring fixes baked in (no action needed from you):
#   - `uv run --extra pushing` syncs pybullet + gym on SLURM compute nodes
#     before the trial starts. Particle jobs SKIP this and use the default
#     dep set; cleanly separated via PEP 621 optional-dependencies.
#   - `--active-env pushing` pins env across training, evaluation, AND log
#     placement. Concurrent edits of config.json can no longer misroute trials.
#
# Layout:
#   pushingA1–A3 : IBC-paper-faithful Langevin recipe, 3 seeds
#                  → variance + sanity check, all in pushing/ folder
#   pushingA4    : A1 + IBC's TRAINING-time Langevin negatives (16 chains,
#                  25 iters) — paper's contrastive negative source
#   pushingA5    : A4 + 20 control points (CP coverage probe)
#   pushingA6    : A4 + 30 control points
#   pushingA7    : A4 + 150k training steps (saturation probe)
#   pushingA8    : A4 + larger MLP Q (mlp 256x4) ablation — does ResNet
#                  actually beat MLP at this size of dataset?
#
# Predictions:
#   - A1 ≈ A2 ≈ A3 within 10pp variance (architecture-only baseline)
#   - A4 > A1 by ≥15pp (training-time Langevin negatives are the key IBC trick)
#   - A5/A6 ≈ A4 (CP cloud size shouldn't dominate at this scale)
#   - A7 > A4 by ≤5pp (paper saturates near 100k)
#   - A8 < A4 (ResNet should win on this dataset size)
