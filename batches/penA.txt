# Batch penA — first pass at D4RL pen-human-v2 (AdroitHandPen-v1, 24-D actions).
#
# Reference: IBC paper (Florence et al. 2021), Table 2 + Appendix D.1.
#   - Reported pen-human-v0 EBM return: 2586 ± 65 (avg over 3 seeds, 100 evals).
#   - Recipe: Langevin EBM, 100k steps, batch 512, LR 5e-4 (decay 0.99/100),
#     512×8 MLP with spectral norm, 8 train counter examples (Langevin),
#     gradient_penalty (margin=1, hinge, final-step-only).
#     Inference: Langevin 100 iters (2nd-iter LR 1e-5).
#
# The IBC paper uses ONLY Langevin counter examples (8 of them).
# Q3CIBC adds a CP generator on top: we want to test whether the CP cloud
# (sampled in 24-D action space, trained jointly with MSE) lets us match or
# beat 2586 with LESS inference compute than 100 Langevin iters/step.
#
# All trials share the IBC paper's architecture:
#   q_network_kind=mlp, q_width=512, q_depth=8, q_use_spectral_norm=true
#   cp_network_kind=mlp, cp_width=512, cp_depth=8
# and the paper's training Langevin hypers (lr_init=0.5, noise=0.5, clip=0.5,
# decay_power=2, lr_final=1e-5, 100 iters). All trials use seed=0; variance
# across seeds is a follow-up (penB) once we know which config to commit to.
#
# Compute notes:
#   - pen-human has 50 demos (~5k transitions). batch=512 → ~10 epochs per
#     10k steps, so 100k steps is ~100 epochs (well past overfitting on MSE
#     alone — but the InfoNCE branch + spectral norm + GP regularize Q).
#   - Training Langevin: 8 negs × 100 iters × forward Q pass per step is
#     expensive (~2-4x baseline per step). 100k steps fits in 25h SLURM.
#   - Inference: 100 Langevin iters × 100 episodes × 200 steps/episode
#     ≈ 2M Q-forward passes per evaluation. Cutting this 4x (A4) or to 0
#     (A5) is the headline "less inference compute" claim.
#
# Things explicitly NOT in this batch (deferred to penB):
#   - Multi-seed reps (the published number is over 3 seeds; we run seed=0
#     here to triage configs, then rerun winners across 3 seeds in penB).
#   - DFO refinement at inference. The paper uses Langevin for D4RL because
#     DFO is known to fail past ~5D, and pen is 24-D. We could still try
#     CP-DFO (inference_dfo_iterations>0) since the CP cloud is a trained
#     starting population — but that's a Q3CIBC-specific bet that we'll
#     park until we have an A1-style ceiling number to compare against.
#   - longer training (150k–200k steps). pen-human is so small that 100k
#     should be saturated; if A1 trains below paper, longer is worth trying.
